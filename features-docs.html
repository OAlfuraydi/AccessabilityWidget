Visual Adjustments

Control how text and colors appear on the page — essential for users with low vision, color sensitivity, or reading difficulties.

🔠
Font Size
fontSize

Description

Scales all text on the page proportionally in 5 progressive steps (100% → 110% → 120% → 130% → 150%). Does not break layouts because scaling is applied via CSS custom properties on the body and respected by responsive designs.

Who Benefits

  • Users with low vision, presbyopia, or aging eyesight
  • Anyone reading on small screens or at a distance
  • Users with mild visual fatigue from prolonged screen time

Effect

Body text, headings, buttons, and labels all grow at a proportional rate. Clicking the feature cycles through 5 levels and then resets.

🔤
Font Family
fontFamily

Description

Replaces the page typeface with one of 4 alternatives: Default (site default), Readable (Atkinson Hyperlegible / high-legibility sans-serif), Dyslexia (OpenDyslexic-style font with weighted bottom letters), or Monospace (fixed-width for code-like clarity).

Who Benefits

  • Users with dyslexia, who benefit from letter-distinctive fonts
  • Users who find decorative site fonts hard to read
  • Users with visual processing differences

Effect

All text across the page switches to the selected font while preserving size and structure.

↕️
Line Height
lineHeight

Description

Increases vertical spacing between lines of text in 3 steps (normal → 1.5× → 2×). Reduces visual crowding and makes it easier to track lines of running text.

Who Benefits

  • Users with dyslexia or reading tracking difficulties
  • Users with attention-related conditions (ADHD)
  • Anyone reading long articles or dense content

Effect

Paragraphs and lists become airier; lines no longer appear cramped, making vertical reading smoother.

🔡
Letter Spacing
letterSpacing

Description

Spreads individual characters apart in 3 steps (normal → 0.05em → 0.1em). Prevents letters from visually merging, which is a common challenge in dyslexia and certain vision conditions.

Who Benefits

  • Users with dyslexia who experience "letter crowding"
  • Users with low vision or astigmatism
  • Users learning a new language or script

Effect

Each letter within every word becomes more distinct, reducing confusion between similar-looking characters (e.g., "rn" vs "m").

⬅️
Text Alignment
textAlign

Description

Forces all body text to left-align (or right-align in RTL/Arabic context). Eliminates "justified" text, which creates uneven word spacing — a well-documented barrier for dyslexic readers.

Who Benefits

  • Users with dyslexia (WCAG strongly recommends avoiding justified text)
  • Users who find uneven spacing distracting

Effect

Paragraphs get even, predictable left/right edges; word spacing becomes uniform.

⬛
High Contrast
highContrast

Description

Applies a black background with bright yellow text for all content. Produces the maximum possible contrast ratio (≈19.5:1), far exceeding WCAG AAA (7:1).

Who Benefits

  • Users with severe low vision or macular degeneration
  • Users with photosensitivity or glare sensitivity
  • Users reading in bright outdoor environments

Effect

Entire page switches to black + yellow. Mutually exclusive with Dark Mode, Light Mode, Grayscale, Invert, and Mute Colors.

🌙
Dark Mode
darkMode

Description

Inverts the lightness of the page: backgrounds become dark, text becomes light. Preserves hue and colors (unlike pure invert), so images and logos remain recognizable.

Who Benefits

  • Users with light sensitivity or migraines
  • Users reading at night or in low-light rooms
  • Users who experience eye strain from bright whites

Effect

Backgrounds turn near-black; text turns near-white. Mutually exclusive with High Contrast, Light Mode, Grayscale, Invert, and Mute Colors.

☀️
Light Mode
lightMode

Description

Forces a bright white background with dark text, overriding any site theme. Useful when a site uses a dark theme that a user finds hard to read.

Who Benefits

  • Users with cataracts (often struggle with dark backgrounds)
  • Users who need maximum text clarity
  • Users taking notes or studying from the page

Effect

White background + black text applied globally. Mutually exclusive with High Contrast, Dark Mode, Grayscale, Invert, and Mute Colors.

◐
Grayscale
grayscale

Description

Removes all color from the page, applying a 100% grayscale CSS filter. Content becomes a pure black-and-white image with preserved brightness relationships.

Who Benefits

  • Users with color vision deficiency (colorblindness)
  • Users distracted by bright colors (ADHD, cognitive sensitivities)
  • Users whose sites convey meaning only via color

Effect

Every element — images, icons, backgrounds — becomes grayscale. Mutually exclusive with the other color-mode toggles.

🔄
Invert Colors
invertColors

Description

Applies a full CSS invert filter to every visible color (e.g., black → white, red → cyan). Different from Dark Mode because it also flips image hues.

Who Benefits

  • Users with severe light sensitivity
  • Users preferring maximum visual differentiation
  • Users with specific visual preferences unmet by dark mode

Effect

Every color shifts to its opposite on the color wheel. Mutually exclusive with other color-mode toggles.

🎭
Mute Colors
muteColors

Description

Desaturates colors by 60%, producing a softer, less intense palette without going fully grayscale. Ideal for reducing visual stimulation without losing color cues.

Who Benefits

  • Users with sensory processing sensitivities or autism
  • Users prone to seizures triggered by saturated colors
  • Users with migraine-related visual triggers

Effect

Colors become muted and pastel-like. Mutually exclusive with other color-mode toggles.

🔗
Highlight Links
highlightLinks

Description

Adds a bright yellow background and thick underline to every hyperlink on the page. Makes interactive elements immediately scannable.

Who Benefits

  • Users with cognitive difficulties identifying clickable elements
  • Users on sites with low-contrast or unconventional link styling
  • Users with low vision who cannot easily spot subtle hover cues

Effect

All <a> elements become visually distinct with yellow highlighting and a bold underline.

🏷️
Highlight Headings
highlightHeadings

Description

Adds a visible background and border to every heading (h1 through h6) on the page, making document structure scannable at a glance.

Who Benefits

  • Users with cognitive differences who benefit from structural cues
  • Users skimming long articles for specific sections
  • Screen-reader users visually verifying heading hierarchy

Effect

Each heading gets a colored background box, giving the page a more structured, outline-like appearance.

📖

Reading Tools

Specialized tools for readers who need visual focus assistance or have learning differences.

🎭
Reading Mask
readingMask

Description

Creates a horizontal "spotlight band" that follows the cursor vertically. The rest of the screen is dimmed via a dark box-shadow overlay, leaving only a narrow strip clearly visible.

Who Benefits

  • Users with ADHD, autism, or attention difficulties
  • Users with reading comprehension challenges
  • Users who lose their place on long pages

Effect

Only a horizontal strip (approximately 100px tall) around the cursor stays lit; the rest fades to dark, forcing visual focus onto one line.

📏
Reading Guide
readingGuide

Description

Draws a thin (2px) colored horizontal line that tracks the cursor, mimicking a physical reading ruler. Helps readers follow a specific line of text without losing place.

Who Benefits

  • Users with dyslexia (line-tracking difficulties)
  • Users with visual perception issues
  • Children and adults learning to read

Effect

A persistent blue line follows the mouse cursor horizontally, acting as a digital reading ruler without hiding any content.

🎯
Focus Mode
focusMode

Description

Dims sidebars, navigation, ads, and non-primary content while keeping the main article or content column fully visible. Reduces distractions from peripheral page elements.

Who Benefits

  • Users with ADHD or attention regulation challenges
  • Users with autism who are overwhelmed by busy layouts
  • Anyone trying to focus on a long article

Effect

Non-essential elements (nav, aside, footer, ads) become ~20% opaque; the main content remains fully visible.

🧠
Dyslexia-Friendly Font
dyslexiaFont

Description

One-click preset that applies a dyslexia-optimized typeface (OpenDyslexic-style) with weighted bottom letters that prevent characters from appearing to "flip" or merge — a common dyslexic reading challenge.

Who Benefits

  • Users with dyslexia (primary use case)
  • Users with broader reading or learning disabilities

Effect

Full page switches to the dyslexia-friendly font. Equivalent to selecting "dyslexia" from the Font Family option, with a dedicated one-click toggle.

⏸️
Pause Animations
pauseAnimations

Description

Stops all CSS animations, transitions, auto-playing videos, GIFs, and carousels on the page. Respects prefers-reduced-motion semantics for a safer experience.

Who Benefits

  • Users prone to seizures (photosensitive epilepsy)
  • Users with vestibular disorders (motion sickness from movement)
  • Users with ADHD distracted by movement

Effect

All motion on the page stops immediately. Videos pause at current frame; carousels freeze; animations halt.

⌨️

Navigation & Motor

Tools for users navigating without a mouse or requiring larger interactive targets.

🖱️
Big Cursor
bigCursor

Description

Replaces the default mouse cursor with a much larger (64×64 px) high-contrast black cursor delivered via inline SVG data URI. Easier to locate and track across the screen.

Who Benefits

  • Users with low vision who lose track of the pointer
  • Users with motor conditions (tremors, Parkinson's) needing larger targets
  • Older users or anyone using a high-resolution display

Effect

The mouse pointer becomes a prominent, easy-to-spot arrow visible across any background.

⌨️
Keyboard Navigation
keyboardNav

Description

Enhances visible keyboard focus styles with a thick, high-contrast outline on every focusable element (buttons, links, inputs) — even if the original site suppresses default focus rings.

Who Benefits

  • Users who cannot use a mouse (motor disabilities)
  • Users who prefer keyboard navigation (power users, screen-reader users)
  • Users on sites that violate WCAG 2.4.7 (Focus Visible)

Effect

Every element reached via Tab/Shift-Tab gets a highly visible blue or yellow outline.

⏭️
Skip to Content
skipContent

Description

Injects a "Skip to main content" link as the first focusable element on the page. Pressing Tab once reveals it, allowing keyboard and screen-reader users to bypass repetitive navigation menus.

Who Benefits

  • Screen-reader users (saves dozens of keystrokes per page)
  • Keyboard-only users with motor disabilities
  • Users on cognitively overwhelming pages

Effect

A "Skip to main content" button appears at the top-left when focused, jumping the user directly to <main> or the primary content region.

🔊

Audio

Text-to-speech and sound control tools that adapt the auditory experience.

🗣️
Text-to-Speech
textToSpeech

Description

Activates click-to-read mode. Once enabled, clicking any text element on the page reads it aloud using the browser's built-in Web Speech API (SpeechSynthesis). Automatically selects Arabic (ar-SA) or English (en-US) voice based on the detected page language.

Who Benefits

  • Users who are blind or have severe visual impairment
  • Users with dyslexia or reading difficulties
  • Non-native speakers learning pronunciation
  • Users multitasking (listen while driving, cooking, etc.)

Effect

When active, a ℹ️ cursor indicates clickable text; clicking triggers speech playback in the page language.

🔇
Mute Sounds
muteSounds

Description

Instantly mutes every audio element, video, and embedded media player on the page by setting their muted property to true.

Who Benefits

  • Users in quiet environments (libraries, offices, public transport)
  • Users with auditory sensitivities or misophonia
  • Users with hearing aids who need consistent sound control

Effect

All media on the page goes silent immediately; playback continues without sound.
