📖 Overview

Insijam is a SaaS accessibility solution that lets website owners embed a fully-featured accessibility toolbar on any website using a single <script> tag. The platform consists of three parts:

⚙️

API Server

Node.js + Express backend. Handles auth, license validation, analytics, and widget config storage.

🛡️

Admin Panel

Internal dashboard for the platform operator to manage customers, subscriptions, and licenses.

🏠

Customer Dashboard

Self-service portal where customers create license keys, embed the widget, and customize branding.

♿

Widget Script

Tiny self-contained JS file that customers paste into their websites. No external dependencies.

📖 نظرة عامة

انسجام هو حل SaaS لإمكانية الوصول يتيح لأصحاب المواقع تضمين شريط أدوات إمكانية وصول متكامل في أي موقع باستخدام وسم <script> واحد فقط. تتكون المنصة من ثلاثة أجزاء:

⚙️

خادم API

خلفية Node.js + Express تتولى المصادقة والتحقق من التراخيص وتخزين التحليلات وإعدادات الودجت.

🛡️

لوحة الإدارة

لوحة تحكم داخلية لمشغّل المنصة لإدارة العملاء والاشتراكات والتراخيص.

🏠

لوحة تحكم العميل

بوابة خدمة ذاتية يمكن للعملاء من خلالها إنشاء مفاتيح الترخيص وتضمين الودجت وتخصيص الهوية البصرية.

⚙️ Server Setup

Prerequisites

  • Node.js 18 or later
  • MySQL 8.0 or later (or MariaDB 10.6+)
1

Create the MySQL database & schema

mysql -u root -p < saas/api/schema.sql

This creates the insijam database with all tables and inserts demo seed data.

2

Configure environment variables

cd saas/api
cp .env.example .env

Edit .env and fill in your values:

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASS=your_mysql_password
DB_NAME=insijam
PORT=3001
ADMIN_SECRET=choose_a_strong_secret_here
ALLOWED_ORIGINS=https://yourdomain.com
🚨

Change ADMIN_SECRET to a long random string before going to production. Anyone with this key has full admin access.

3

Install dependencies & start

npm install
npm start          # production
# OR
npm run dev        # development (auto-restart on changes)

The server starts on http://localhost:3001

4

Verify it's running

curl http://localhost:3001/api/v1/health

Expected response: {"status":"ok"}

🔐 Admin Login

Open http://localhost:3001/admin in your browser.

1

Enter the Server URL

Default: http://localhost:3001. On a production server, use your domain (e.g. https://api.yourdomain.com).

2

Enter the Admin Secret Key

This is the value of ADMIN_SECRET from your .env file.

3

Click "Login"

The panel verifies the key by calling GET /api/v1/admin/stats. On success you see the admin dashboard.

⚠️

The admin key is stored only in sessionStorage — it is cleared when you close the browser tab. You will need to log in again next session.

🔐 تسجيل دخول المسؤول

افتح http://localhost:3001/admin في متصفحك.

1

أدخل رابط الخادم

الافتراضي: http://localhost:3001. على خادم الإنتاج، استخدم نطاقك (مثال: https://api.yourdomain.com).

2

أدخل مفتاح المسؤول السري

هو القيمة التي وضعتها في ADMIN_SECRET من ملف .env.

3

انقر "تسجيل الدخول"

تتحقق اللوحة من المفتاح عبر GET /api/v1/admin/stats. عند النجاح تظهر لوحة الإدارة.

⚠️

يُخزَّن مفتاح الإدارة في sessionStorage فقط — يُحذف عند إغلاق تبويب المتصفح. ستحتاج لتسجيل الدخول في كل جلسة جديدة.

📋 Managing Subscriptions

Each customer has exactly one subscription record. The Subscriptions tab lets you change plan, status, website limit, and expiry date.

Subscription Statuses

StatusMeaningWidget Behavior
trialNew account, not yet paidWidget works normally (within license limits)
activePaid and currentWidget works normally
expiredPast expiry dateWidget validation will fail — widget shows error
suspendedManually suspendedWidget validation will fail — widget shows error

How to Edit a Subscription

1

Find the customer

In the Subscriptions tab, find the row by customer name or email.

2

Click the Edit (✏️) button

A form opens with current values pre-filled.

3

Change values and Save

You can change: Plan (starter/professional/enterprise), Status, Website Limit (number), and Expiry Date.

✅

Changing the plan also automatically updates the plan field on all existing licenses belonging to that customer.

🔑 Managing Licenses

Each license key is tied to one customer and one domain. The Licenses tab shows all keys across all customers.

ActionEffect on Widget
Set status = activeWidget on that domain works normally
Set status = suspendedWidget immediately stops working — shows "service suspended" error
Set status = expiredWidget shows "license expired" error
Delete licenseWidget shows "invalid license key" error on next page load
ℹ️

License status changes take effect immediately because the widget re-validates on every page load (result is cached in sessionStorage per tab, not across tabs).

💰 Billing

The Billing tab shows all payment records. Billing records are created manually or by your payment integration (e.g. Stripe webhook). Each record contains:

  • Customer — who paid
  • Invoice ID — your reference (e.g. INV-2026-001)
  • Plan — what plan was purchased
  • Amount — stored in cents (e.g. 7900 = $79.00)
  • Status — paid / pending / failed

💰 الفواتير

يعرض تبويب الفواتير جميع سجلات المدفوعات. تُنشأ سجلات الفواتير يدوياً أو عبر تكامل الدفع (مثل Stripe webhook). يحتوي كل سجل على:

  • العميل — من دفع
  • رقم الفاتورة — مرجعك (مثال: INV-2026-001)
  • الخطة — ما الخطة التي اشتُريت
  • المبلغ — مخزون بالسنت (مثال: 7900 = 79.00$)
  • الحالة — مدفوع / معلق / فاشل

🔑 License Keys

Each website that uses your widget needs its own license key. License keys are generated automatically when you add a site.

Add a new site

1

Go to "My Sites"

Click the 🌐 My Sites section in the sidebar.

2

Click "Add Site"

Enter the domain of your website (e.g. mystore.com) and an optional display name.

3

Copy your license key

A key in the format AW-XXXX-XXXX-XXXX-XXXX is generated. Copy it — you will paste it into your website's HTML.

⚠️

Each license key is domain-specific. A key issued for mystore.com will only work on pages served from mystore.com. It will fail on a different domain.

Suspend / reactivate a site

Click the toggle button next to any site to suspend or reactivate it. Suspension takes effect immediately — the widget on that site stops working.

Delete a site

Click the 🗑 button. The license key is permanently deleted and the widget on that site will show an error on the next page load.

🌐 Embedding the Widget

Add this snippet to every page of your website, just before the closing </body> tag:

<script>
  window.AccessibilityWidgetConfig = {
    licenseKey: 'AW-YOUR-LICENSE-KEY-HERE',  // ← paste your key
    lang:       'auto',                       // 'en', 'ar', or 'auto'
    position:   'bottom-right',               // bottom-right, bottom-left, top-right, top-left
  };
</script>
<script src="https://yourserver.com/widget/accessibility-widget.js"></script>

What happens on page load

  1. The widget calls POST /api/v1/validate with your license key and the current domain.
  2. The server checks: Is the key valid? Is the domain correct? Is the subscription active?
  3. If valid → widget loads, your branding settings are applied, and an analytics event is recorded.
  4. If invalid → a small error notice appears at the bottom of the page; the widget does not load.
✅

The validation result is cached in sessionStorage so it only makes one API call per browser tab, not on every page navigation.

  };
</script>

ما يحدث عند تحميل الصفحة

  1. يستدعي الودجت POST /api/v1/validate بمفتاح الترخيص والنطاق الحالي.
  2. يتحقق الخادم: هل المفتاح صالح؟ هل النطاق صحيح؟ هل الاشتراك نشط؟
  3. إذا كان صالحاً → يحمّل الودجت وتُطبَّق إعدادات هويتك البصرية ويُسجَّل حدث تحليلي.
  4. إذا كان غير صالح → تظهر إشعار خطأ صغير في أسفل الصفحة ولا يحمّل الودجت.
✅

نتيجة التحقق مخزنة في sessionStorage فيُجري استدعاء API واحداً فقط لكل تبويب متصفح، وليس عند كل تنقل في الصفحة.

🛠️ Development Mode

When building or testing your site locally (e.g. http://localhost:8080), the widget cannot validate against the server because your local domain is not registered. Use devMode: true to skip validation:

<script>
  window.AccessibilityWidgetConfig = {
    devMode: true,   // ← skips license check
    lang: 'auto',
  };
</script>
<script src="accessibility-widget.js"></script>
🔒

devMode is domain-locked. The widget checks window.location.hostname at startup. If the page is not served from localhost, 127.0.0.1, or a file:// URL, the devMode flag is ignored and a real license is required. You cannot bypass validation by copying the script to your site and flipping devMode: true.

🛠️ وضع التطوير

عند بناء أو اختبار موقعك محلياً (مثال: http://localhost:8080)، لا يستطيع الودجت التحقق من الخادم لأن نطاقك المحلي غير مسجل. استخدم devMode: true لتخطّي التحقق:

<script>
  window.AccessibilityWidgetConfig = {
    devMode: true,   // ← يتخطى التحقق من الترخيص
    lang: 'auto',
  };
</script>
<script src="accessibility-widget.js"></script>
🔒

devMode مقفل على النطاق. تتحقق الأداة من window.location.hostname عند البدء. إذا لم تكن الصفحة من localhost أو 127.0.0.1 أو رابط file://، يُتجاهل devMode ويُطلب ترخيص حقيقي. لا يمكن تجاوز التحقق بنسخ السكريبت إلى موقعك وتفعيل devMode: true.
