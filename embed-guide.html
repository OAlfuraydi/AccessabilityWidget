Quick Start

Add the widget to any page with a single script tag. Place it just before the closing </body> tag.

Minimal Embed

<!-- Place before </body> -->
<script src="accessibility-widget.js"></script>

With Configuration

<!-- 1. Configure before loading -->
<script>
  window.AccessibilityWidgetConfig = {
    lang: 'en',
    position: 'bottom-right',
    primaryColor: '#2563EB'
  };
</script>

<!-- 2. Load widget -->
<script src="accessibility-widget.js"></script>
💡 The widget auto-initializes as soon as the DOM is ready. No further JavaScript is needed on your end.

Configuration Reference

Set window.AccessibilityWidgetConfig before loading the widget script. All properties are optional.

PropertyTypeDefaultDescription
langstring'en'Widget language: 'en', 'ar', or 'auto' (detects from <html lang>)
positionstring'bottom-right'Trigger button position: 'bottom-right', 'bottom-left', 'top-right', 'top-left'
primaryColorstring'#2563EB'Hex color used for the button, toggles, and highlights
buttonIconstring'♿'Icon or text displayed inside the floating trigger button
storageKeystring'aw_settings'localStorage key for persisting user preferences
showProfilesbooleantrueShow accessibility profile presets at the top of the panel
showTTSbooleantrueShow the text-to-speech section in the panel
showReadingMaskbooleantrueInclude reading mask feature in the panel
featuresobject{all: true}Object to individually enable or disable specific features (see Feature Control)

Feature Control

Any feature can be disabled by setting it to false in the features config object. Disabled features are completely hidden from the widget panel.

Framework Integration

React / Next.js

// In _app.tsx or pages/_document.tsx
import { useEffect } from 'react';

export default function App({ Component, pageProps }) {
  useEffect(() => {
    window.AccessibilityWidgetConfig = {
      lang: 'auto',
      primaryColor: '#2563EB'
    };
    const script = document.createElement('script');
    script.src = '/accessibility-widget.js';
    script.async = true;
    document.body.appendChild(script);
    return () => document.body.removeChild(script);
  }, []);
  return <Component {...pageProps} />;
}

Vue / Nuxt.js

// In nuxt.config.js
export default {
  head: {
    script: [
      { src: '/accessibility-widget.js', defer: true, body: true }
    ]
  }
}

// Or in app.vue
mounted() {
  window.AccessibilityWidgetConfig = { lang: 'auto' };
  const s = document.createElement('script');
  s.src = '/accessibility-widget.js';
  document.body.appendChild(s);
}

Accessibility Profiles

Six built-in profiles activate multiple settings simultaneously. Users can click any profile to apply it, or click again to reset. Profiles can be disabled by setting showProfiles: false in config.

👁️ Visually Impaired

fontSize: 2, highContrast: true, highlightLinks: true, bigCursor: true, keyboardNav: true

🔍 Low Vision

fontSize: 3, fontFamily: 'readable', lineHeight: 2, highContrast: true, highlightLinks: true, highlightHeadings: true

📖 Dyslexia Support

fontFamily: 'dyslexia', lineHeight: 2, letterSpacing: 2, textAlign: 'left', highlightLinks: true

🛡️ Seizure Safe

pauseAnimations: true, muteColors: true, lightMode: true

🧠 Cognitive Support

fontFamily: 'readable', lineHeight: 1, focusMode: true, readingGuide: true, pauseAnimations: true, textAlign: 'left'

⌨️ Keyboard Navigation

keyboardNav: true, skipContent: true, highlightLinks: true

JavaScript API

The widget instance is available at window._accessibilityWidget after the script loads. You can also instantiate it manually for advanced control.

MethodDescription
.open()Open the accessibility panel programmatically
.close()Close the accessibility panel
.toggle()Toggle the panel open/closed
.resetAll()Reset all settings to defaults and remove all applied classes
.applyProfile(name)Apply a profile by name: 'visuallyImpaired', 'dyslexia', 'seizureSafe', etc.
.toggleTTS()Start or stop text-to-speech reading
.toggleLang()Switch widget interface language between English and Arabic
.destroy()Remove the widget from the DOM and reset all styles

Usage Examples

// Access the running instance
const widget = window._accessibilityWidget;

// Open the panel from a custom button
document.getElementById('my-a11y-btn').addEventListener('click', () => {
  widget.open();
});

// Apply a profile programmatically
widget.applyProfile('dyslexia');

// Create a new instance with custom config
const myWidget = new AccessibilityWidget({
  lang: 'ar',
  position: 'bottom-left',
  primaryColor: '#059669',
  showProfiles: false
});

Arabic & RTL Support

The widget provides full support for Arabic-language and bilingual websites. When set to Arabic, the widget panel switches to RTL layout with Arabic labels.

Language Detection

Setting lang: 'auto' makes the widget read the <html lang> attribute. If it starts with ar, the Arabic interface is used; otherwise English.

Manual Language Setting

⚠️ The language toggle button inside the panel always allows users to switch between Arabic and English regardless of the default setting.
